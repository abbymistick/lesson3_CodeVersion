Highlight #1 Version Control

Version control is a key component to creating any project. It allows you to save different versions, or milestones, of a project so that you can go back and compare versions against each other. You can also revert an entire project back to a certain version. There are both local and centralized locations where you can store your versions. Local is easier to access quickly; however, when versions are in a centralized lcoation then it allows many users to access the files and edit them. This allows teams to work on projects together easily.

Highlight #2 What Is Git?

Git is essentially a system that stores the different versions of your files. When you create a new version of a file, git will recognize that something has changed and will store what the new version of the file looks like. If git notices that nothing has changed then it will not save anything new, instead it will just link back the latest version. Git mainly uses a local drive so there is no need for other computers or systems in order for git to work effectively.

Highlight #3 Repository

A repository stores all of the saves snapshots of various versions of a project. In the repository there is a staging area. This is where all of the files that have been edited, or changed, are located before they are committed to the local repository. Once something is committed to the repository, the snapshot is saved.
